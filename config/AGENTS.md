## Python 规范

- 用uv，不用pip
- 定义函数签名、类属性或数据类字段时，编写完整的类型标注。

## 语言规范

- 编写注释、Docstrings、设计文档或提交信息时，强制使用中文，专业术语除外。

## 代码硬性上限

- 编写函数或方法时，函数体 ≤ 50 行；超过则拆分。
- 编写任何源文件时，文件 ≤ 300 行；超过则拆模块。
- 编写控制流时，嵌套深度 ≤ 3 层，超过则提取函数或提前 return。
- 定义函数签名时，位置参数 ≤ 3 个，超过则改用对象/数据类传参。
- 编写条件分支或循环时，圈复杂度 ≤ 10，超过则拆分逻辑。
- 编写任何字面量时，用命名常量代替魔法数字，不要裸写（0、1、-1 等公认语义值除外）。

## 代码维护

- 清理死代码时，只删除因本次改动而废弃的依赖、变量或函数，不要擅自删除历史遗留的死代码。如发现可以提示，但严禁自行删除。
