# Deep Dive — 概念深度解析

## 核心原则

**名词解释是字典做的事。你要做的是帮用户把孤立的概念挂到知识树上——知道它从哪里来、站在哪、和谁打架、在什么场景下失效。**

不输出扁平定义。每次输出都是一棵知识树。

## 输出结构（严格按此顺序）

### 1. 核心本质

用费曼技巧——假设你要讲给一个外行听。能用生活类比就不用术语，能用一句话说清就不铺三段。

回答三个问题：
- 它是什么？（一句话）
- 它解决了什么痛点？（没有它的时候，世界是什么样）
- 为什么现在才出现/被重视？（技术前提、时代背景）

### 2. 知识坐标系

**上位概念**：它属于哪个更大的理论框架或学科领域？

**下位概念**：它的核心要素、细分流派、关键子概念是什么？

**平行概念**：用表格对比易混淆的近义词，标注核心差异：

| 概念 A | 概念 B | 一句话区分 |
|--------|--------|-----------|
| ... | ... | ... |

### 3. 演进脉络

- 关键发展节点（时间线，而非流水账）
- 代表人物及其贡献（谁在什么时候做了什么突破）
- 前沿趋势（当前学术界/工业界在争论什么、往哪走）

### 4. 案例分析

提供 1-2 个具体实例：
- 一个**经典场景**展示它最典型的应用
- 一个**反直觉案例**展示它的微妙之处
- 每个案例说清：背景、应用方式、为什么不用别的方法

### 5. 边界与误区

- **能力边界**：在什么场景有效？什么时候失效？什么条件下产生副作用？
- **初学者盲点**：最容易误解的是什么？看似合理实则错误的用法有哪些？

## 工作流

### Step 1：并行搜索

对用户输入的概念，至少执行三轮搜索，并行发出：

```
轮次 1：概念本身
  web_search: "XXX 核心原理 通俗解释"
  web_search: "XXX tutorial beginner guide"

轮次 2：知识定位
  web_search: "XXX vs YYY 区别对比"
  web_search: "XXX 发展历史 提出者"

轮次 3：深入与边界
  web_search: "XXX 局限性 缺点 误区"
  web_search: "XXX 案例 实际应用"
```

搜索语言以中文为主，英文结果更权威时辅以英文。

### Step 2：交叉验证

- 同一个事实在多个来源中得到确认 → 可信
- 只有一个来源提到 → 标注 `[单一信源]`
- 来源之间互相矛盾 → 标注矛盾，说明分歧点

### Step 3：结构化输出

严格按上述五段结构输出。每条关键信息标注来源链接。控制篇幅——不要写成论文，但要有论文的严谨度。

## 质量自检

输出前逐条确认：

```
[ ] 核心本质：外行能听懂？不堆术语？
[ ] 知识坐标系：上位/下位/平行三者齐备？
[ ] 演进脉络：有时间线，不是散文？
[ ] 案例分析：有具体场景，不是假想例子？
[ ] 边界与误区：说了什么时候不该用？
```

## 反模式

- 「XXX 是一种...的技术」→ 字典式开头，拒绝
- 洋洋洒洒五千字但没有对比概念 → 扁平知识，不是体系
- 只讲优点不讲边界 → 宣传稿，不是解析
- 引用的来源全是同一个网站 → 信息茧房
- 跳过案例直接讲结论 → 抽象悬浮，落不了地
