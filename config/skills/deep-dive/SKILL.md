# Deep Dive — 概念深度解析

## 核心原则

用费曼技巧把孤立概念挂到知识树上——讲清来源、定位、对比和边界，而非扁平定义。

## 输出结构（严格按此顺序）

### 1. 核心本质

费曼式讲解给外行听。回答三个问题：
- 它是什么？（一句话）
- 解决什么痛点？（没有它时世界什么样）
- 为什么是现在才出现？（技术前提、时代背景）

### 2. 知识坐标系

- **上位概念** —— 所属理论框架
- **下位概念** —— 核心要素、细分流派
- **平行概念** —— 易混淆概念对比：`概念 A vs 概念 B —— 一句话区分`

### 3. 演进脉络

- 关键发展节点（时间线）
- 代表人物及其贡献
- 前沿趋势（当前争论与方向）

### 4. 案例分析

1-2 个实例：
- 经典场景 + 反直觉案例
- 每个说清：背景、应用方式、为何选它

### 5. 边界与误区

- **能力边界** —— 何时有效/失效/副作用
- **初学者盲点** —— 最常见误解与错误用法

## 工作流

### Step 1：并行搜索

三组并行：
1. 概念本身 —— 核心原理、通俗解释、tutorial/beginner guide
2. 知识定位 —— 对比区别、发展历史与提出者
3. 深入边界 —— 局限性与误区、案例与实战

中文为主，英文权威时辅之。

### Step 2：交叉验证

多源确认 → 可信；单一来源 → 标注 `[单一信源]`；矛盾 → 标注分歧。

### Step 3：结构化输出

按五段结构输出，关键信息标注来源。控制篇幅，保持严谨。

## 质量自检

- [ ] 外行能听懂
- [ ] 上位/下位/平行齐备
- [ ] 有时间线
- [ ] 有具体案例
- [ ] 说明失效场景

## 反模式

- 字典式开头（「XXX 是一种…的技术」）
- 无对比概念的扁平知识
- 只讲优点不讲边界
- 单一信源
- 跳过案例只讲结论
