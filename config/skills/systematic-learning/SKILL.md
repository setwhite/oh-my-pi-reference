# 体系化学习——知识体系构建

## 核心原则

用费曼技巧把孤立概念挂到知识树上——讲清来源、定位、对比和边界，而非扁平定义。

## 写作风格

- 语气直接、克制。像同事写内部 wiki
- 禁止套路过渡语：不说「众所周知」「不言而喻」「值得注意的是」
- 禁止逢迎语气：不说「希望对你有帮助」「欢迎留言讨论」
- 禁止贴标签式概括：不说「外壳 vs 灵魂」「两层声音」这类 slogan
- 允许「注意」「换句话说」「对比来看」等引导语，克制使用
- 引用优于转述——给出原始出处让读者自行验证
- 论点必须有具体例子或代码片段支撑，禁止纯抽象描述

## 输出格式

直接生成 HTML 知识卡片，写到当前工作目录，命名 `YYYY-MM-DD-标题.html`。

HTML 规范：
- 完全自包含，不依赖任何 CDN 或外部资源
- 字体使用系统栈，不加载 web font
- 设计服务于阅读，不为炫技。无渐变、无 box-shadow 堆叠、无 backdrop-filter
- 响应式：移动端可读

## 输出结构（严格按此顺序）

### 1. 核心本质

费曼式讲解给外行听。回答三个问题：
- 它是什么？（一句话）
- 解决什么痛点？（没有它时世界什么样）
- 为什么是现在才出现？（技术前提、时代背景）

### 2. 知识坐标系

- **上位概念** —— 所属理论框架
- **下位概念** —— 核心要素、细分流派
- **平行概念** —— 易混淆概念对比：`概念 A vs 概念 B —— 一句话区分`

### 3. 演进脉络

- 关键发展节点（时间线）
- 代表人物及其贡献
- 前沿趋势（当前争论与方向）

### 4. 案例分析

1-2 个实例：
- 经典场景 + 反直觉案例
- 每个说清：背景、应用方式、为何选它

### 5. 边界与误区

- **能力边界** —— 何时有效/失效/副作用
- **初学者盲点** —— 最常见误解与错误用法

## 工作流

### Step 1：并行搜索

三组并行：
1. 概念本身 —— 核心原理、通俗解释、tutorial/beginner guide
2. 知识定位 —— 对比区别、发展历史与提出者
3. 深入边界 —— 局限性与误区、案例与实战

### Step 2：交叉验证

- 多源确认 → 可信
- 单一来源 → 标注 `[单一信源]`
- 来源矛盾 → 标注分歧，列出双方依据与出处

### Step 3：生成 HTML 卡片

按五段结构生成 HTML，关键信息标注来源。控制篇幅，保持严谨。

## 质量自检

- [ ] 外行能听懂
- [ ] 上位/下位/平行齐备
- [ ] 有时间线
- [ ] 有具体案例
- [ ] 说明失效场景

## 反模式

- 字典式开头（「XXX 是一种…的技术」）
- 无对比概念的扁平知识
- 只讲优点不讲边界
- 单一信源
- 跳过案例只讲结论
