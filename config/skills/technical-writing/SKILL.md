# 技术文章写作

## 核心准则

- 不要只是堆信息量，要把文章写得条理清晰，方便读者理解
- KISS，用简短的语言讲清楚，减少重复啰嗦
- 尽量符合输出结构
- 不懂就联网搜索，再不懂就问用户
- 论点要有具体例子或证据，禁止靠抽象描述堆砌，禁止编造
- 学会用引用代替转述，保持文章主干干净，同时给读者可验证的出口

## 输出结构

```
# 标题（一句话讲清帮读者解决什么问题）

> **撰写日期**：YYYY 年 M 月 D 日

## 一、省流
1-2句话，这是读者的初印象，决定读者会不会继续看下去

## 二、正文
一级标题到四级标题的编号：一、 → （一） → 3. → （4）
表格更适合对比/罗列，mermaid适合复杂逻辑关系的图，按需选择

## 三、总结
小于3句话，这是读者的最终印象，决定读者会不会点赞和互动

## 参考链接
(如果有就加上)
```

## 工作流

### 一、获取信息
网络搜索工具，搜官方文档、github仓库、博客

### 二、搭骨架
按输出结构出大纲，先不写正文
大纲发给用户，用户确认后再开始生成

### 三、写正文
按大纲展开

### 四、冗余自检
有没有重复表达？
表格里的内容是否必要？不必要就该用段落，不要加抽象描述
如果有代码块做示例，尽量使用行内注释，不再另建参数说明表
